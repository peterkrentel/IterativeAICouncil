# AI Council - Deployment Guide

Complete guide for deploying AI Council to AWS EC2 + K3s using GitHub Actions.

## Architecture

```
GitHub Actions (CI/CD)
    ↓
AWS EC2 (t3.micro - Free Tier)
    ↓
K3s (Lightweight Kubernetes)
    ↓
AI Council API (FastAPI)
    ↓
Free LLM APIs (Groq + Gemini)
```

## One-Time Setup (Run Locally)

### Step 1: Configure AWS CLI

```bash
aws configure
# Enter your AWS Access Key ID
# Enter your AWS Secret Access Key
# Default region: us-east-1
# Default output format: json
```

### Step 2: Run Bootstrap Script

This creates S3 bucket, DynamoDB table, and OIDC provider for GitHub Actions:

```bash
cd /path/to/IterativeAICouncil
chmod +x scripts/bootstrap.sh
./scripts/bootstrap.sh
```

The script will output GitHub secrets. **Save these!**

### Step 3: Add GitHub Secrets

Go to: `https://github.com/YOUR_USERNAME/IterativeAICouncil/settings/secrets/actions`

Click "New repository secret" and add each of these:

**AWS Secrets (from bootstrap script output):**
- `AWS_REGION` - e.g., `us-east-1`
- `AWS_ROLE_ARN` - e.g., `arn:aws:iam::123456789012:role/GitHubActionsRole`
- `TF_STATE_BUCKET` - e.g., `aicouncil-terraform-state-1234567890`
- `TF_LOCK_TABLE` - e.g., `aicouncil-terraform-locks`

**LLM API Keys:**
- `GROQ_API_KEY` - Get from https://console.groq.com
- `GOOGLE_API_KEY` - Get from https://aistudio.google.com/app/apikey

**SSH Key (will be generated):**
- `EC2_SSH_PRIVATE_KEY` - Will be generated by Terraform (see below)

### Step 4: Create Production Environment

Go to: `https://github.com/YOUR_USERNAME/IterativeAICouncil/settings/environments`

1. Click "New environment"
2. Name: `production`
3. Add protection rules:
   - ✅ Required reviewers (add yourself)
   - ✅ Wait timer: 0 minutes
4. Save

## Deployment Workflows

### Workflow 1: Terraform Plan (Automatic)

**Trigger:** Automatically runs on PR to `terraform/` directory

**What it does:**
- Validates Terraform code
- Generates infrastructure plan
- Comments plan on PR

**No action needed** - just review the plan in PR comments.

### Workflow 2: Terraform Apply (Manual)

**Trigger:** Manual via GitHub Actions UI

**What it does:**
- Creates AWS infrastructure:
  - VPC, subnet, security groups
  - EC2 instance with K3s
  - ECR repository
- Outputs EC2 IP and SSH key

**How to run:**
1. Go to Actions → "Terraform Apply"
2. Click "Run workflow"
3. Type `apply` in the confirmation field
4. Click "Run workflow"
5. Approve in the production environment
6. Wait ~5 minutes for completion

**After completion:**
1. Download the `terraform-outputs` artifact
2. Extract `ssh_private_key` from the JSON
3. Add it as GitHub secret: `EC2_SSH_PRIVATE_KEY`

### Workflow 3: Build and Deploy (Automatic)

**Trigger:** Automatically on push to `main` branch (or manual)

**What it does:**
- Builds Docker image
- Pushes to ECR
- Deploys to K3s cluster
- Runs health checks

**How to trigger manually:**
1. Go to Actions → "Build and Deploy"
2. Click "Run workflow"
3. Select branch: `main`
4. Click "Run workflow"

### Workflow 4: Destroy (Manual Only)

**Trigger:** Manual via GitHub Actions UI

**What it does:**
- Destroys ALL AWS infrastructure
- ⚠️ **WARNING:** This is irreversible!

**How to run:**
1. Go to Actions → "Destroy Infrastructure"
2. Click "Run workflow"
3. Type `destroy` in the confirmation field
4. Click "Run workflow"
5. Approve in the production environment

## Complete Deployment Steps

### First Time Deployment

1. **Run bootstrap script** (locally, one time)
   ```bash
   ./scripts/bootstrap.sh
   ```

2. **Add GitHub secrets** (from bootstrap output)

3. **Create production environment** (GitHub settings)

4. **Deploy infrastructure:**
   - Go to Actions → "Terraform Apply"
   - Type `apply` and run
   - Wait for completion
   - Download outputs artifact
   - Add `EC2_SSH_PRIVATE_KEY` to GitHub secrets

5. **Deploy application:**
   - Go to Actions → "Build and Deploy"
   - Run workflow
   - Wait for completion

6. **Test deployment:**
   ```bash
   curl http://YOUR_EC2_IP/health
   ```

### Subsequent Deployments

Just push to `main` branch:

```bash
git add .
git commit -m "Update application"
git push origin main
```

GitHub Actions will automatically build and deploy!

## Accessing Your Deployment

### API Endpoint

```bash
# Health check
curl http://YOUR_EC2_IP/health

# API docs
open http://YOUR_EC2_IP/docs
```

### SSH to EC2

```bash
# Save the private key
echo "$EC2_SSH_PRIVATE_KEY" > k3s-key.pem
chmod 600 k3s-key.pem

# SSH to instance
ssh -i k3s-key.pem ubuntu@YOUR_EC2_IP
```

### Access Kubernetes

```bash
# Get kubeconfig
scp -i k3s-key.pem ubuntu@YOUR_EC2_IP:~/.kube/config ./kubeconfig

# Use kubectl
export KUBECONFIG=./kubeconfig
kubectl get pods
kubectl logs -f deployment/aicouncil
```

## Troubleshooting

### Workflow fails with "role cannot be assumed"

**Fix:** OIDC provider not set up correctly
```bash
# Re-run bootstrap script
./scripts/bootstrap.sh
```

### Terraform apply fails

**Fix:** Check AWS credentials and permissions
```bash
aws sts get-caller-identity
```

### Deployment fails with "connection refused"

**Fix:** K3s may still be installing. Wait 5 minutes and retry.

### Application not responding

**Fix:** Check pod logs
```bash
kubectl logs -f deployment/aicouncil
```

## Cost Optimization

### Use Spot Instances (70% savings)

Edit `terraform/variables.tf`:
```hcl
variable "use_spot" {
  default = true
}
```

### Scale Down When Not Using

```bash
# Stop instance (keeps data)
aws ec2 stop-instances --instance-ids YOUR_INSTANCE_ID

# Start instance
aws ec2 start-instances --instance-ids YOUR_INSTANCE_ID
```

### Destroy When Done

```bash
# Run "Destroy Infrastructure" workflow
# Or manually:
cd terraform
terraform destroy
```
